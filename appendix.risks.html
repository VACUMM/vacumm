3.4. Unstable routines¶

This section lists the modules, functions, classes and methods whose stability is little or not proven. The user is cautioned as to their uses.

Warning

This list is probably not exhaustive or updated because there is no direct link between the technical development of a code, and the update of this part of the documentation. It is likely that you can glean some information about the stability of the code in its own documentation.
